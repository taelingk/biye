# Contributing Guide

本仓库面向 CardioFit 项目研发，提交代码前请遵循以下约定。

## 1. 开发原则

- 所有公共函数必须包含类型注解
- 使用 Google 风格 docstring
- 使用 `logging`，禁止 `print`
- 路径处理统一使用 `pathlib.Path`
- 参数从配置文件读取，避免硬编码魔数

## 2. 数据与实验规范

- 数据集按受试者划分，禁止跨集合泄露
- 数据增强只在训练集启用
- 标准化器仅在训练集拟合
- 训练输出需记录模型配置、随机种子、指标

## 3. 分支与提交建议

- 分支命名：`feature/*`、`fix/*`、`refactor/*`
- Commit 建议使用祈使句，突出变更目的
- 每次 PR 尽量聚焦一个模块或一类问题

## 4. 测试要求

- 核心逻辑必须有单元测试
- 提交前至少执行：

```bash
pytest tests/ -v
```

## 5. 代码评审关注点

- 是否引入数据泄露风险
- 是否满足生理合理性约束（输出范围、变量关系）
- 是否破坏多模态时间对齐
- 是否记录可复现实验条件
